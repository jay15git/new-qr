# QR Studio Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the starter screen with a responsive shadcn-based QR generator page that exposes the main `qr-code-styling` controls, renders a live preview, supports logo upload or URL, and exports PNG/JPEG/WEBP/SVG.

**Architecture:** Keep `app/page.tsx` as a Server Component that renders the page shell and a single `QrStudio` client entry point. Put QR editor defaults and the state-to-library transformation in a focused helper module so preview/download logic can reuse the same options shape and be covered with tests.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Tailwind CSS v4, shadcn/ui radix-nova components, qr-code-styling, Vitest.
