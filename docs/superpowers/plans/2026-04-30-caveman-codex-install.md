# Caveman for Codex Installation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install the `JuliusBrussee/caveman` Codex plugin and enable repo-local auto-start for this workspace without disturbing the existing Codex environment setup.

**Architecture:** Caveman has two separate pieces in Codex. The plugin install gives Codex the `$caveman` command. Repo-local auto-start is a second layer implemented through `.codex/config.toml` and `.codex/hooks.json`, which must exist in the target repository. This repo already has `.codex/environments/environment.toml`, so the install should add new files alongside it and leave the autogenerated environment file untouched.

**Tech Stack:** Codex desktop plugin system, repo-local `.codex` configuration, GitHub-hosted Caveman plugin repository

### Task 4: Enable Repo-Local Auto-Start in `new-qr`

**Files:**
- Create or merge: `.codex/config.toml`
- Create or merge: `.codex/hooks.json`
- Preserve untouched: `.codex/environments/environment.toml`

- [ ] **Step 1: Add the Codex feature flag**

Create `.codex/config.toml` with:

```toml
[features]
codex_hooks = true
```

Expected:
- Codex hook support is enabled for this repository

- [ ] **Step 2: Add the repo-local SessionStart hook**

Create `.codex/hooks.json` with:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup|resume",
        "hooks": [
          {
            "type": "command",
            "command": "echo 'CAVEMAN MODE ACTIVE. Rules: Drop articles/filler/pleasantries/hedging. Fragments OK. Short synonyms. Pattern: [thing] [action] [reason]. [next step]. Not: Sure! I would be happy to help you with that. Yes: Bug in auth middleware. Fix: Code/commits/security: write normal. User says stop caveman or normal mode to deactivate.'",
            "timeout": 5,
            "statusMessage": "Loading caveman mode"
          }
        ]
      }
    ]
  }
}
```

Expected:
- Caveman auto-starts when Codex opens or resumes inside `new-qr`

- [ ] **Step 3: Merge carefully if either file already exists**

Rules:
- If `.codex/config.toml` already exists, merge the `[features]` table instead of replacing the file
- If `.codex/hooks.json` already exists, append the Caveman `SessionStart` hook instead of overwriting unrelated hooks
- Do not edit `.codex/environments/environment.toml`

Expected:
- Existing repo-local Codex behavior stays intact
